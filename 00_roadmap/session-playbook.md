# Session playbook (works even with unpredictable days)

## 15 minutes
- Read 1 KB page (from `10_kb/`) OR skim your own solution notes
- Do 1 tiny drill: re-implement a template (two pointers / BFS / binary search) from memory

## 30–45 minutes
- Solve 1 problem (prefer “Core Track”)
- Log it in `30_registry/problems.csv` (status + takeaway)

## 60–90 minutes
- Solve 2 problems OR 1 timed problem
- Update tracker:
  - mark weak ones as `revisit` and set `next_review`

## Hard rule
If you solved something but didn’t write a 1–2 line takeaway in the tracker,
you’ll likely waste time later re-learning it.
