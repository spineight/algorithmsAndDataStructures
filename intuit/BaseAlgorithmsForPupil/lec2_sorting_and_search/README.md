### Двоичный поиск

#### Общая идея
Рассмотрим концепцию двоичного поиска, которая более общая,
которая в частности позволяет нам найти и элемент в массиве и решить другие задачи.

Пусть у вас есть массив и есть какое-то утверждение про его элементы `A(a[m])`
И пусть выполняется следующее, сначала про какое-то множество элементов эта функция 0,
а потом с некоторого места(возможно никакого) 1.  
   000000000000000/11111111111  
a |############################|  
Вот например в поиске элементов в массиве мы в качестве этой функции выбрали:  
`A(a[m])=[a[m]>=x]`  
В обозначении Кнута это значит, что функция вернет 1(True) если `a[m] >= x`, иначе 0(False)


Наш двойчный поиск будет искать первое место в массиве где 1.
Мы ищем первый элемент в массиве для которого функция возвращает 1 (либо обнаружим что все нули)
1. Интервал на котором мы ищем, удобно брать (L,R]  
Так как:
if a[m] >= x --> идем налево и правый конец включаем(так как m-ый элемент нам подходит) - (L,m]  
else --> идем направо и m-ый элемент не включаем(он нам не подоходит) - (m, R]  
2. Идея виртуальных элементов - для того, чтобы выполнялся инвариант цикла
Добавим один элемент -inf, который точно меньше искомого x в начало
И один элемент +inf, который точно больше искомого x  
Так как у нас инвариант, что длина промежутка на котором мы ищем = (R-L) > 1   
На этом промежутке больше одного элемента, то мы никогда не спрашиваем значения `a[L], a[R]`

a [-inf][ | | | | | | | | | | ] [+inf]

##### Инвариант цикла:
ответ лежит в промежутке (L,R]   

Длина промежутка R-L, пока длина промежутка > 1 мы ищем.
Осталось два момента:  
1. Для L : c чего начать (какой элемент нам точно не подходит), добавляют в начале виртуальный
элемент который точно не подходит.
2. Для R - какой элемент нам точно подоходит, заведем еще один элемент

Когда длина интервала >=2 мы никогда не спрашиваем про L и R элемент,
поэтому то, что L или R за границами массива нас устраивает
Берем:
L = -1
R = n

Какой конец включать, какой нет - мы считаем, что `a[L] = -inf, a[R] = +inf`  
Соответственно, так как и в начале цикла инвариант должен выполняться, то например:  
первый элемент < x - `[L,R)` - так как -inf точно меньше
первый элемент > x - `(L,R]` - так как +inf точно больше

Когда цикл закончится, интервал будет длины 1, и искомый элемент будет в R, так как наш инвариант (L,R]

```c++
while(r-l>1) {
  m = (l+r)/2;
  if(a[m] >= x)
    r=m;
  else
    l=m;
}
if(r<n) {// found элемент >= x
  if(a[r] == x)
    return r;  // found
}
```

#### Задача
Для монотонной функции вида `F(x)= X^3 + ax + b = с`, где `a > 0`, `b > 0`, `c > 0`  
найти такой первый целый x: `что x^3+ax+b=c`
Возвращает -1 при неудаче.  
Заметим, что a>0 , b > 0  
При возрастании x => F(x) возрастает    
Давайте виртуально(в уме нашем, заведем массив, назовем его f и забьем в него
 f(0), f(1)...f(n)  
 Заметим, что в этом массиве значения будут возрастать    
 Предположим то, что мы добьемся, что f(n) > c    
 f(k) < c  
 Нам нужно найти такой элемент i, что f(i) = c

#### Вещественный двоичный поиск
Пусть есть функция, которая возрастает и вы хотите найти минимальное место(x0), где эта функция
больше чем какое-то C.   
Удвоением левой и правой границы выберем интервал (см. Универсальный метод поиска значений L,R),
так, чтобы x0 точно попало в этот интервал    
Вопрос до скольки сужать интервал.  
Есть три метода метода:  
* Пока длина промежутка не станет меньше Epsilon (10^-9), это значит что X0  
вы определили, он лежит на этом промежутке где-то  
* Смотреть на разность значений функций на L и R, когда они близки, это значит, что
c уже практически найдено  
У этих обоих способов проблема (вещественных чисел бесконечно много, а в компьютере они хранятся в конечной длины слове)
`double` - 8 byte  
Так может случится, что если ответ большой, то нет в этом районе таких чисел которые отличиются меньше, чем
на 10^-9, тогда двоичный поиск зациклится

Проблема может быть что L,R - два соседних по представлению вещественных числа

##### Способ 3(Абсолютно точный вещественный двоичный поиск)  
Давайте искать до тех пор, пока у нас не окажется два соседних по представлению вещественных числа.  
Утверждается, что два вещественных числа соседние по представлению, если середина интервала между ними,
равна одному из них   

##### Еще более простой метод

##### Полная анатомия дерева
Дерево - есть вершины(узлы) и у каждого узла есть несколько детей,
в двоичном дереве не более двух.  
Корень - самая верхняя вершина    
Вершины у которых есть дети - внутренние  
Лист - вершина без детей  
Пень - дерево из одной вершины  

У дерева есть высота - расстояние от корня до самого далекого листа  
У каждой вершины есть две характеристики - высота(расстояние от нее до самого глубокого листа у нее в подддерве)  
и глубина - расстояние от нее до корня   
У каждой вершины есть родитель - это тот к кому она непосредственно подвешана  
Все, что лежит на пути от нас до корня - наши предки(включая нас)  
Все что у нас лежит в поддереве (включая нас) - потомки  
Мы изучили подвешеное дерево (оно подвешенное за корень)
Бывает еще свободное дерево - когда корень не выделен

Нас будут интересовать двоичные деревья с дополнительным условием:
глубины листьев отличаются не более чем на 1,
слои заполняются слева на право. Такое дерево называется почти полное двоичное дерево.  

Куча - почти полное двоичное дерево, у которого в узлах хранятся какие-то значения(ключи),
выполнено следующее - ключ родителя <= ключа детей


За счет того, что куча почти полное двоичное дерево - ее можно упаковать в массив.  
Упаковка: занумеровываем вершины по слоям.  
Давайте теперь разбиремся как обратно это дело распаковывать:  
для вершины `i` дети: `2*i, 2*(i+1)` - если первая вершина имеет индекс = 1  
Посмотрим почему это так:  
у нас в каждом следующем слое в x2 раза больше вершин, чем в предыдущем  
Если мы занумеруем первый слой, то во втором нам понадобится в 2 раза больше чисел
1  
2  
4  
По индукции можно доказать совсем математически строго  
Чтобы найти родителя `i` вершины: `i/2`  

`a [i div 2] <= a[i]` - условие кучи

В середине алгоритма начальный кусок - куча, хвостик - хвост будующего отсортированого массива

### Quick sort Быстрая сортировка (сортировка Хоара)
На практике в 2-3 раза быстрее сортировки кучей  

Рассечение по X , выберем некоторый элемент массива и сделаем так, что:  
```
|      <= x     | x |   >=x       |
 l             j     i           r
```
Отсортируем тем же способом левую и правую части массива
Утверждается, что весь массив отсортируется  

Мы сортируем куски массива, поэтому нашей процедуре нужны индексы от куда и до куда сортировать
```c++
void quick_sort(std::vector<int> &a, int l, int r)
```

Рассечение:  
идем двумя указателями, один назавем i, другой j  так же есть число X
наша цель это чтобы слева от i оставались только  элементы <= x  
а справа от j >= x

Есть массив есть указатель i, есть указатель j
```c++
           i              j
```  
Инвариант следующий(Чтобы понять что ничего не зациклится):  
все элементы до i <= x  
все элементы  после j >= x    
есть элемент >= x  после i (хотябы один ) и есть элемент <= x до j (хотябы один)


#### Время работы
В худщем случае за O(n^2)  
Если берем случайный элемент, то можно доказать, что в среднем время работы будет O(nlogn)
