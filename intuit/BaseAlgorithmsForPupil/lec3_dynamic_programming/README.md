## Динамическое программирование
Это такой способ решать задачи где надо посчитать кол-во чего-нибудь,
или найти оптимальный путь какой-нибудь.  

Саму задачу мы решать не умеем, но мы умеем ее превратить ее в несколько более маленьких подзадач
, каждую из которых решить, а потом из этих ответов (решений) собрать ответ нашей основной задачи.  
Но это еще не совсем динамическое программирование, просто рекурентный подход переход к меньшей задаче.  
Динамическим программированием становится тогда, когда мы замечаем, что вообще всего возможных
постановок задач у нас не так много и все возможные меньшие задачи, которые бы может надо решить,
мы решаем, вообще все и запоминаем их ответы и одну задачу два раза не решаем.  

### Числа Фибонначи
```
F1=1, F2=1   
n>=3 Fn = F{n-1} + F{n-2}
```

```c++
int fib(int n) {
  if(n == 1 || n == 2) return 1;
  return fib(n-1) + fib(n-2); 
}
```

```    
                5
           /         \
        4             3
      /   \           / \
    3       2        2   1
   /\                
  2  1              
```
Можно показать, что время выполнения этой программы ~ `1.5^n`

Обратите внимание, третее число мы считаем два раза, вызовы функции Fib не знают друг о друге ничего,
поэтому каждое из них он вычисляет заново.  
Разумным представляется возможность запомнить ответы на предыдущие задачи.  
Нам понадобятся задачи каких-то меньших n для решения задачи fib(n)
Нам понадобятся `fib(n-1), (n-2), (n-3), (n-4)...`  
На самом деле нам надо знать `F1,F2...Fn`, вместо того, чтобы вычислить только `Fn`, мы все равно вычислим `F1,F2...Fn`  
Давайте не будем мучаться и все их по очереди вычислим  
Вместо рекуксивной функции, которая сразу следует из определения, но долго работает мы
напишем вместо этого вычисление по порядку,которое будет работать быстро.  
Заведем массив от 1 до `MaxN` и его заполним
```c++
vector<int> fib(n+1);
fib[1]=fib[2]=1;
for(int i = 3; i < n; ++i)
  fib[i] = fib[i-1] + fib[i-2]; // (1) 
```
Обратите внимание у нас рекуррентная формула, которой мы пользовались, она снова есть, но на этот раз,
каждое значение будет вычислено ровно один раз

Вот в числах Фибонначи мы применили такую идею, хорошо
рассмотрим еще одну задачу в которой аналогичная идея применяется.
  
### Задача про кузнечика
Пусть у вас есть дорога, которую затопило, но каждый метр остался столбик, затопило, но врехушки столбиков видны,
и на нулевом столбике сидит кузнечик. Он умеет прыгать вперед на 1 и 2, он хочет допрыгать до n.
Как же нам найти количество путей, которыми мы можем допрыгать до точки n?
Идея:  
посмотрим на предпоследний столбик на котором мы были,  
перед тем как прыгнули на n-ый  
Если кузнечик прыгает на 1,2 - мы могли быть перед этим на n-1 и могли быть на n-2   
Какими-то способами мы могли припрыгнуть на n-2 столбик и какими-то другими на n-1   
Если обозначить число способов как a, то получается рекурентное соотношение:  
`a[n] = a[n-1] + a[n-2]`  
Нужно разобраться с чего начать  вычисления, мы видим что для `a[1]` уже рекурентное соотношение наше не выполнено
(`a[-1]`- нету, минус первого столбика не бывает, значит нужно отдельно рассмотреть нулевой и минус первый столбик)
Сколько способов припрыгнуть с нулевого столбика до нулевого (вот ничего не делать сколько способов) -
ничего не делать есть один способ (сидеть и ничего не делать)
`a[0] = 1`  
`a[1] = 1` 

## Этапы (Общая схема решения задачи с помощью динамического программирования)
Что нужно делать, когда решаешь задачу на динамическое программирование:
1. Что мы считаем(какие значения мы вычисляем)  (пример `a[n]` - кол-во способов допрыгать до n столбика)
2. Как их пересчитывать, Какое рекурентное соотношение на значения, которые мы считаем (`a[n] = a[n-1] + a[n-2]`)
Когда есть рекуррентная формула она позволяет вычислять значения через меньшие.
3. Какие начальные значения (тоесть какие значения не подходят под общую схему рекурентной формулы и поэтому их надо отдельно указать) `a[0] = 1 a[1] = 1`  
Иногда при выбранном рекурентном соотношении сложно задать начальные значения, поэтому приходится менять рекурентное соотношение
4. В каком порядке вычислять значения (в задаче про кузнечика это очевидно)
Когда мы начали вычислять числа Фибоначи мы перешли от вычисления с конца, когда мы запускаемся рекурсивно к вычислению сначала,
когда мы вычисляем по очереди: первое, второе,третье ... благодаря этому нам удалось использовать уже существующие значения,
их дважды не пересчитывать и поэтому быстро вычислять
5. Где искать ответ  
Для кузнечика: `a[n]`

Если нам надо не просто посчитать оптимальное значение или количество чего-то,  
но еще и как этого минимума достичь, то у нас есть достаточно универсальный способ:
нужно в одном, иногда двух вспомогательных массивах запоминать с помощью чего достигается max/min  
Для вывода ответа: вспомогательный массив, например from,
мы получим последовательность значений приводящую нас к оптимальному решению, если она в обратном порядке,тоесть
мы используем либо свой стэк, либо рекурсивную процедуру (с программным стэком)

Важный принцип Оптимальность для подзадач:  
Мы можем использовать ДП - только если у нас есть оптимальность для подзадач.  
 - чтобы нашу задачу решить оптимально, нужно предыдущие задачи,  
какие-то более меньшие тоже решить оптимально.
