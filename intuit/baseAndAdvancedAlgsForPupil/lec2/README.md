Когда мы делаем сортировку - мы ищем перестановку элементов,
в каком порядке нужно элементы переставить, чтобы массив отсортировался

В `void countingSort(vector<uint32_t> &a)` мы сортировали числа.
Обычно когда решается задача о сортировке есть какой-то ключ x, по которому мы сортируем
и какие-то полезные данные : `(x,data`
Нам нужно не потерять данные.

В случае если мы бы хотели применить тот же алгоритм:
для каждого x посчитали сколько раз встречается (x у нас от 0 до С) и выписали бы x
Сами x нам выписать недостаточно, нам нужно еще каждому элементу сопоставлять данные.

Если у нас все x от 0 до C, то отсортированный массив можно разбить на блоки:
```c++
// |   |   |   | ... |    |
//  0    1   2         C-1
```
В каждом блоке будут все пары (x,data) - у которых x == индексу блока
`num[c]` - длина блока с индексом c,
зная длины блоков мы для каждого блока `i` можем вычислить индекс начала `pos[i]`

```c++
//    |        |        |       | ... |       |
//       0          1       2            C-1
//    |        |        |             |
//   pos[0]    pos[1]   pos[2]        pos[C-1]
```

```c++
pos[i] = pos[i-1] + num[i-1];
```
Теперь мы знаем позиции и можно сделать следующее: можно еще раз пройтись по нашему массиву пар `(x,data)`
По x мы получаем блок в котором эта пара должна лежать, и запихиваем пару именно в этот блок, на позицию начала блока,
а позицию начала блока сдвигаем на единицу, в каждом блоке у нас сколько-то ячеек уже заполнено и `pos[i]` - 
указывает на первую свободную ячейку блока `i`
Единственное что эти пары нужно записывать не в тот же массив, а в новый - мы пока бежим по одному массиву портить его не имеем права.

У нас хранятся два массива: a[], data[]

Мы идем по массиву, встретили пару `(a[i], data[i])`
нам этот элемент нужно куда-то в новом массиве положить, мы его кладем на позицию `pos[a[i]]`
```c++
aNew[pos[a[i]]] = a[i]
dataNew[pos[a[i]]] = data[i]
``` 
Это если мы хотим в явном виде получить новый массив, в парвильном порядке упорядоченный.
На самом деле, когда мы решаем задачу о сортировке мы ищем не сам переупорядоченный массив  а  перестановку элментов,
в каком порядке нужно элементы переставить, чтобы массив отсортировался, тогда можно просто говорить, что в случае если
мы будем искать перестановку `P`
```c++
P[pos[a[i]]] = i
```
Причем вот таким способом сортировку часто писать удобнее,
например если данные какие-то большие их неудобно копировать, в случае если у нас ключи `a[i]` по которым мы сравниваем
маленькие, например целые числа можем сравнивать за `O(1)`, при этом данные которые мы за собой таскаем могут быть
достаточно большие, строчки длиной 1000 символов каждая, нам приходится их копировать,
естественно это делать не надо в случае если мы хотим получить просто перестановку
```c++
P[pos[a[i]]] = i
```
Когда мы получаем просто перестановку мы за `O(1)` находим нужный индекс, данные нигде не копируются

Сортировка называется устойчивой когда данные `data[i]` которые шли с одинаковым ключем(которые мы сравнивать не умели
не умели определить какое из них должно идти раньше) в отсортированном массиве
будут в таком же порядке как и в исходном. 
Устойчивая сортировка не меняет порядок одинаковых элементов

Сортировка подсчетом которую мы написали - устойчивая, благодаря ее устойчивости мы сможем изобрести цифровую сортировку

DigitalSort, RadixSort, Поразрядная сортировка
Пускай мы хотим отсортировать строки длины 2 за `O(n)`
Сторки длины 1 за `O(n)` отсортировать можно подсчетом (каждый символ у нас char - `[0,255]`,`c= 256`)
```c++
|_|_|
|_|_|
|_|_|
|_|_|
|_|_|
|_|_|
```
Сперва отсортировать их по последнему символу, а затем по первому
Когда мы будем сортировать по первому символу - для тех строчек у которых первый символ равен - будет задан
верный порядок по второму символу, они уже правильно отсортированы по второму символу.

В общем случае для `K` символов если все строки имеют длину K мы только что научились их сортировать за время `O(nK))`
Дополнительной памяти на требуется всего `O(C)` - где C - размер алфавита
По скольку ответом является перестановка нам еще нужно `O(n)` доп памяти для хранения перестановки
Мы считаем перестановку, строчки никуда не копируем, потом если нам очень хочется мы их перекопируем и переставим в нужном порядке

Как сортировать целые числа?
Любое целое 32 битное число можно представить в виде 4 симвллов каждый по `2^8`
Получится строка из 4 символов, сортируем как строчки
